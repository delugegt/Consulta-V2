Gerenciamento de Processos no LINUX

Você pode enviar sinais que encerram processos; existem uma variedade de tipos de sinais que se correlacionam exatamente como processo é tratado pelo kernel.

Como iniciar os processos?

Vamos começar falando sobre namespaces. O Sistema Operacional (OS) usa namespaces para dividir os recursos disponíveis no computador para processos (como CPU, RAM e prioridade). O processo com um ID de 0 é um processo que é iniciado quando o sistema inicial.

Obtendo processos/serviços para iniciar no boot

Algumas aplicações podem ser iniciadas no boot do sistema que possuímos. Dizer ao servidor web apache para iniciar apache manualmente e, em seguida, dizendo ao sistema para lançar apache2 no boot.

Comandos Importantes:

ps - Podemos usar o comando ps para fornecer uma lista dos processos em execução como sessão do nosso usuário e algumas informações adicionais, como: código de status; sessão que está executando-o; quanto tempo de uso da CPU está usando; nome do programa ou comando real que está sendo executado;

ps aux - Para ver os processos executados por outros usuários e aqueles que não funcionam a partir de uma sessão (ou seja, processos de sistema): Fornecer aux ao comando os. para podermos ver "root" e "cmnatic"

top - fornece estatísticas em tempo real sobre os processos em execução em seu sistema em vez de uma visualização única. Essas estatísticas se atualizarão a cada 10 segundos.

Kill - Para finalizar um comando, podemos usar o comando de kill apropriadamente nomeado e o PID associado. Ou seja, para matar pid 1337, usaríamos kill 1337. Sinais que podemos enviar para um processo quando ele é morto:

  • SIGTERM - Mate o processo, mas permita que ele faça algumas tarefas de limpeza de antemão
  • SIGKILL - Mate o processo - não faz nenhuma limpeza após o fato
  • SIGSTOP - Pare/suspenda um processo

systemctl - este comando nos permite interagir com o processo/daemon sistemd. Continuando com nosso exemplo, o systemctl é um comando fácil de usar que leva a seguinte formatação: systemctl [opção] [serviço]. Para dizer ao apache para iniciar, vamos usar systemctl start apache2. Podemos fazer quatro opções com systemctl:

  • Start
  • Stop
  • Enable
  • Disable
